Here you can find an automatically generated reference for ProPhyle’s CLI.

General¶

prophyle (list of subcommands)¶

$ prophyle  -h

usage: prophyle.py [-h] {init,index,classify} ...

positional arguments:
  {init,index,classify}
                        sub-command help
    init                Initialize data
    index               Create index
    classify            Classify reads

optional arguments:
  -h, --help            show this help message and exit

prophyle init¶

$ prophyle init -h

usage: prophyle.py init [-h] [-m DIR] <library>

positional arguments:
  <library>                genomic library ['bacteria', 'viruses', 'plasmids',
                           'hmp', 'all']

optional arguments:
  -h, --help               show this help message and exit
  -m DIR, --prophyle-dir DIR
                           ProPhyle directory [/home/simone/prophyle]

prophyle index¶

$ prophyle index -h

usage: prophyle.py index [-h] -n FILE -l DIR [-t INT] [-k INT] [--continue]
                         <index.dir>

positional arguments:
  <index.dir>             index directory (will be created)

optional arguments:
  -h, --help              show this help message and exit
  -n FILE, --newick FILE  taxonomy tree (in Newick format)
  -l DIR, --lib-dir DIR   directory with genomic sequences
  -t INT, --threads INT   number of threads [auto=4]
  -k INT, --kmer-len INT  k-mer length [32]
  --continue              continue with index construction (construct only
                          missing parts)

prophyle classify¶

$ prophyle classify -h

usage: prophyle.py classify [-h] [-k INT] [-n] [-m {h1,c1}] [-o {kraken,sam}]
                            [--annotate] [--tie-lca] [--mimic-kraken]
                            <index.dir> <reads.fq>

positional arguments:
  <index.dir>              index directory
  <reads.fq>               file with reads in FASTA or FASTQ [- for standard
                           input]

optional arguments:
  -h, --help               show this help message and exit
  -k INT, --kmer-len INT   k-mer length [32]
  -n, --no-klcp            do not use k-LCP
  -m {h1,c1}, --measure {h1,c1}
                           measure: h1=hit count, c1=coverage [h1]
  -o {kraken,sam}, --out-form {kraken,sam}
                           output format
  --annotate               annotate assignments
  --tie-lca                use LCA when tie (multiple hits with the same
                           score)
  --mimic-kraken           mimic Kraken algorithm and output (for debugging
                           purposes)
